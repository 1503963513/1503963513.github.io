    http缓存

    卓儿 6/4/2022 http

    # http缓存又分为强缓存和协商缓存

    # 强缓存

    • 当我们第一次发起http请求的时候：

    需要加载整个资源文件，列如：css、html、js、image ......

    • 第一次之后请求

    流程是：CPU计算、页面渲染、网络请求

    前两歩其实都特别快、主要是网络请求有包的大小的影响

    # 强缓存上怎么实现的呢

    • 第一次请求，返回资源：响应头内设置一个cache-control

    max-age：缓存的时间

    no-cache：不缓存

    我们浏览器下一次发起请求的时候，如果max-age没过期，我们从本地缓存中拿资源，这就是强缓存

    如果max-age过期，会再次发起请求

    （其实强缓存不需要我们前端做任何操作，因为是从respone响应头来决定的）

    # 协商缓存

    协商缓存主要是对资源版本的资源标识

    # 理解：

    • 第一次请求：服务器返回资源以及资源标识，并存入本地缓存。
    • 后续请求：请求服务器带着本地缓存的标识，如果本地缓存的标识和服务器最新版不一样，服务器返回200状态码，会在次发送资源以及标识，存入本地缓存；如果是最新版本，服务器返回304状态码，直接从本地缓存中拿取

    # 资源标识

    ETag：上一串字符串（推荐使用）

    last-Modified：用时间来作为标识，精确到秒级

    # 总结：

    当无缓存的情况下：

    判断是否有缓存————>服务器返回资源————>渲染页面

    协商缓存有缓存的情况下

    判断是否有缓存————>是否标识过期

    过期就重新返回资源，没有过期就从本地缓存拿取
