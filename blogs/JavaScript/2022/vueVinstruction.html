    Vue指令详解

    卓儿 2/12/2022 vue

    # 指令

    指令是在 html 标签内部使用到的，以 v- 开头的特殊属性，指令值（属性值）是 JS 表达式.

    指令的职责是，当表达式的值改变时，将其产生的连带影响，响应式地作用于 DOM。

    • **v-html：**渲染 html 文本(类似于 .innerHTML)

      <span v-html="html"></span>
      1

    • **v-text：**渲染普通文本(类似于 .innerText / .textContent)

      <span v-text="html"></span>
      1

    • **v-bind：**动态绑定属性值，可简写为 :

      $ <div v-bind:title="title">正确提示</div>    
$ <div :title="title">缩写提示</div>
      1
      2

    • **v-on：**绑定事件（注册事件监听），可简写为 @

      $ <button v-on:click="userClickEvent">点击</button>
$ <button @click="userClickEvent">点击</button>
      1
      2

    # 条件渲染：

    • **v-show：**操作的是 CSS 的 display

      <div v-show="condition">不同的是带有 v-show 的元素始终会被渲染并保留在 DOM 中。v-show 只是简单地切换元素的 CSS property display。</div>
      1

    • **v-if：**操作的是 DOM 节点的销毁与重建

      <div v-if="condition">v-if 指令用于条件性地渲染一块内容。这块内容只会在指令的表达式返回 truthy 值的时候被渲染。</div>
      1

    • v-else-if

      x <div v-else-if>表示 v-if 的“else if 块”。可以链式调用。</div>
      1

    • v-else

      <div v-else>Oh ~ NO 我上else块</div>
      1

      v-if 和 v-else-if 和 v-else ：类似于js中的 if(){ }else if(){ } else{ }

    v-if VS v-show（面试）：

    • v-if 操作的是节点的销毁与重建，是“惰性”的（初始条件不满足，不会渲染）
    • v-show 操作的是 CSS 的切换，不管初始条件是什么，元素总是会被渲染

    一般来说，v-if 有更高的切换开销，而 v-show 有更高的初始渲染开销。因此，如果需要非常频繁地切换，则使用 v-show 较好；如果在运行时条件很少改变，则使用 v-if 较好。

    # 列表渲染：

    • **v-for：**循环遍历数组各元素或对象各属性(类似于 for...in/for...of)
    <li v-for="item in array"></li>
<li v-for="(item, index) in array"></li>

<li v-for="value in object"></li>
<li v-for="(value, key) in object"></li>
<li v-for="(value, key, index) in object"></li>

<li v-for="n in number"></li>
    1
    2
    3
    4
    5
    6
    7
    8

    item - 数组中遍历到的当前元素

    index - 元素对应下标

    value - 对象中遍历到的当前属性值

    key - 对象中遍历到的当前属性名

    n in number - 表示固定循环 number 次，n 表示从 1 递进到 number

    # 注意，在语法中使用的 in 也可以替换为 of 的使用，意思完全一致

    1. x 当 v-if 与 v-for 一起使用时（在同一个节点上使用），v-for 具有比 v-if 更高的优先级。不推荐同时使用 v-if 和 v-for。

    2. x 中 v-if 比 v-for 优选级更高。

    • **v-model：**在表单控件或者组件上创建双向绑定。

    它负责监听用户的输入事件以更新数据，并对一些极端场景进行一些特殊处理。

    用法：☟

    注意：v-model 在内部为不同的输入元素使用不同的 property 并抛出不同的事件

    列入: ☟ （这是不用v-model实现双向数据绑定的方法）

    text 和 textarea 元素使用 value property 和 input 事件；

    checkbox 和 radio 使用 checked property 和 change 事件；

    select 字段将 value 作为 prop 并将 change 作为事件。

    以text举例：

    <input type="text" v-bind:value="message" v-on:input="ckMessage">
    1
    • **v-pre：**跳过pre所在的这个元素和它的子元素的编译过程。可以用来显示原始 Mustache 标签。跳过大量没有指令的节点会加快编译。

    用法： ☟

    $ <input type="text" v-model="message" v-pre>
$ <span v-pre>{{ message }}</span>
    1
    2
    • v-once：只渲染元素和组件一次。随后的重新渲染，元素/组件及其所有的子节点将被视为静态内容并跳过。这可以用于优化更新性能。

    用法： ☟

    $ <input type="text" v-model="message" v-once>
$ <span v-once>{{ message }}</span>
    1
    2
    • **v-cloak：**与 CSS 样式 [v-cloak] {display: none;} 一起使用，可以隐藏未编译的模板，当关联实例结束编译后在对应节点上删除 v-cloak 则显示出编译后的渲染结果。

    用法： ☟ （属性选择器配合使用）不会显示，直到编译结束。

    [v-cloak]{ display: none; }
    1
    $ <div v-cloak> {{ message }} </div>
    1
